変更・提案のリスクを洗い出します。担当Role: Architect。

## 読み込む文書

1. 対象範囲のディレクトリのREADME.md(概要/責任/責任外)を読みます
2. `docs/decisions/README.md`一覧を確認し、関連する過去判断があれば個別ファイルを読みます

## 手順

1. 対象範囲(実装・設計・運用のどこに影響するか)を特定します
2. 次の観点でリスクを分類します
   - 後方互換性・既存動作への影響
   - 秘密情報・権限・外部公開への影響
   - 依存追加・更新の要否
   - 手戻りが大きい判断(設計方針の変更等)かどうか
3. 各リスクに深刻度(Critical/High/Medium/Low)と根拠を付けます
4. Critical/Highがある場合は、着手前に意思決定レポートでPMへ確認します
