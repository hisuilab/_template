## 1. 背景と問題

`features/ai-agent` Part（`.claude/` scaffold）をどのプロファイルに含めるかを決定する必要があった。
このテンプレートリポジトリ自体が AI エージェント駆動開発を前提として設計されており、生成した
プロジェクトでも同様のワークフローを採用することが想定される。

## 2. 検討した選択肢

| # | 選択肢 |
| --- | --- |
| A | 全プロファイルに無条件で含める |
| B | `--feature ai-agent` フラグでオプトイン |
| C | `purpose/` または `scale/` によって自動決定 |

## 3. 採用案

**A: 全プロファイルに無条件で含める**（`small-cli` / `small-web-api` / `small-library`）

## 4. 理由

- このテンプレートの主要ユーザーは AI エージェント駆動開発を実践する開発者であり、
  オプトアウトが必要なケースが少数である
- オプトイン機構（`--feature` フラグ）の実装はウィザードと generator の複雑度を増す
- 生成物への影響は `.claude/` ディレクトリのみで、プロジェクトに害を与えない

## 5. トレードオフと影響

- Claude 以外のツールを使う開発者や AI エージェントを使用しないプロジェクトに
  不要なファイルが配布される（`.claude/` を削除するだけで対処可能）
- 将来的にプロファイル多様化が進んだ段階で、オプトイン化を再検討する余地がある
