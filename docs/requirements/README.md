# docs/requirements

## 1. 概要

プロジェクトの要件定義文書を置くディレクトリです。「何を作るか」の正本です。

## 2. 責任

- 利用者・問題・提供価値・対象範囲・成功条件を定義します（`product.md`）
- 機能要件・品質要件・利用者ワークフローは `scale/medium` 以上のプロジェクトで追加します

## 3. 責任外

- 実現方式（`docs/architecture/` が持ちます）
- Issue 単位の設計提案（`docs/design/` が持ちます）
- マイルストーン進捗（`docs/milestones/` が持ちます）
