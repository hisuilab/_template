# docs/domain/model

## 1. 概要

集約ルートの一覧と境界を定義するディレクトリです。
集約ごとに `<aggregate-name>.md` を追加していきます。

## 2. 責任

- 集約ルートの一覧と集約間の関係図
- 各集約の仕様ファイル（`<aggregate-name>.md`）

## 3. 責任外

- ビジネスルール・不変条件（`../rules/` が担当）
- 実装詳細（`src/domain/` が担当）

## 集約ファイルの書き方

`<aggregate-name>.md` を作成し、以下の項目を記述します。

```markdown
# <集約名>

## 責務

何を表す集約か（1〜2行）

## 属性

| 属性 | 型 | 説明 |
| --- | --- | --- |
| id | UUID |  |

## 不変条件

- （この集約が常に満たすべき条件）

## ライフサイクル

作成 → ... → 完了 / 削除

## 他集約との関係

- <集約名>: 参照 / 所有
```
